# 单例模式

**定义:**  保证一个类仅有一个实例，并提供一个访问它的全局访问点。

**类型:** 创建型模式。

**角色及其职责:**  
    1、 单例类（Singleton）：单例类。 

**适用场景:**  
    用于只允许创建一个对象。

**优点:**  
    只有一个实例，减少内存开销；<br>
    设置全局访问点，严格控制访问方式。

**缺点:**  
    没有接口，扩展困难。

**重点:**  
    私有构造方法；<br>
    延迟加载；<br>
    线程安全；<br>
    序列化反序列化安全；<br>
    反射安全。
